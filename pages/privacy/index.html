2. Instagram ZIP과 분석 결과

선택한 ZIP은 현재 브라우저 메모리에서 읽습니다. 팔로워·팔로잉 JSON 원문, 사용자 아이디 전체 목록과 관계 분석 결과를 맞팔체커 서버, 문의 데이터베이스, 뉴스레터 데이터베이스 또는 수요조사 데이터베이스로 전송하지 않습니다.

  • Instagram 아이디·비밀번호, 인증 코드 또는 로그인 토큰을 요구하지 않습니다.
  • 결과는 ZIP 생성 시점의 스냅샷이며 현재 상태와 다를 수 있습니다.
  • A·B 비교 자동화 대상은 팔로워 이탈 계정 중 최신 시점에도 내가 팔로우 중인 계정만 브라우저에서 계산합니다.

3. Chrome Companion 자동화

자동화는 선택 기능이며 웹 분석만 사용할 수도 있습니다. 사용자가 목록을 선택하고 확인란을 누른 경우에만 아이디 목록이 웹 페이지와 설치된 확장 프로그램 사이의 브라우저 메시지로 전달됩니다.

  • 확장 프로그램 권한은 로컬 저장소, 탭 이동, Chrome 사이드패널과 맞팔체커·Instagram 사이트 접근으로 제한합니다.
  • Instagram에 이미 로그인된 탭에서 현재 프로필 경로와 화면에 표시된 팔로잉·팔로우 취소 버튼의 텍스트를 확인합니다.
  • 쿠키, 세션 토큰, 비밀번호, 인증 코드 또는 Instagram 비공개 API를 읽거나 호출하지 않습니다.
  • 작업 목록과 성공·실패 상태는 Chrome 확장 프로그램의 로컬 저장소에 남으며 라바랩스 서버로 전송되지 않습니다.
  • 사이드패널을 닫거나 중지하면 자동으로 다시 시작하지 않습니다.

4. 서비스 문의

문의 페이지에서 이용자가 직접 제출한 정보만 저장합니다. 접수 시 고유 접수번호를 발급하고 담당자 이메일로 알림을 보냅니다.

  • 문의자는 이름, 답변 이메일, 문의 유형, 제목과 내용을 입력합니다.
  • 운영자는 관리자 화면에서 문의 내용, 알림 상태와 처리 상태를 확인할 수 있습니다.
  • ZIP 원본, Instagram 비밀번호, 인증 코드와 복구 코드를 문의 내용에 입력하지 않도록 안내합니다.

5. 뉴스레터 신청과 수신 해지

뉴스레터는 회원가입이나 결제가 아닌 선택 기능입니다. 개인정보 수집·이용과 이메일 수신에 동의한 경우에만 등록됩니다.

  • 구독자마다 임의 UUID 수신 해지 토큰을 발급합니다.
  • 메일의 해지 링크에는 이메일 주소 대신 토큰만 포함합니다.
  • 해지 링크 또는 이메일 입력 방식으로 구독 정보와 연결 동의 기록을 삭제할 수 있습니다.

6. 프리미엄 수요조사와 캠페인

프리미엄 수요조사는 뉴스레터 동의와 분리됩니다. 뉴스레터 캠페인을 대기열에 넣으면 해당 시점의 활성 구독자 이메일과 수신 해지 토큰을 캠페인별 전달 목록으로 저장합니다. Instagram ZIP, 분석 결과와 자동화 목록은 포함하지 않습니다.

8. 제3자 제공과 판매 금지

라바랩스는 개인정보를 판매하거나 일반 광고 목록으로 제공하지 않습니다. 별도 동의, 서비스 제공에 필요한 위탁 또는 법령상 요청이 있는 경우에만 필요한 범위에서 처리합니다.

9. 보유 기간과 삭제

자동화 목록

새 목록을 전송하면 기존 목록을 교체합니다. Chrome 확장 프로그램 관리 화면에서 Companion의 저장 데이터를 삭제하거나 확장 프로그램을 제거하면 로컬 목록과 상태가 삭제됩니다.

서비스 문의

문의 목적 달성 뒤 운영상 필요가 없으면 삭제합니다. 삭제 요청은 문의 페이지에서 접수할 수 있습니다.

뉴스레터와 수요조사

수신 해지 페이지와 수요조사 화면에서 각각 삭제할 수 있습니다.

브라우저 작업 기록

작업공간 관리 또는 브라우저 사이트 데이터 삭제 기능으로 제거할 수 있습니다.

10. 자동 저장 기술과 관리자 인증

웹은 로컬 저장소와 서비스 워커 캐시를 사용합니다. Companion은 chrome.storage.local에 자동화 목록과 설정을 저장합니다. 관리자 화면은 이메일 매직링크를 사용하며 인증 토큰은 해당 브라우저 탭의 세션 저장소에 보관합니다.

11. 이용자의 권리

이용자는 서버에 저장된 본인 정보의 열람, 정정, 삭제와 처리 정지를 요청할 수 있습니다. 웹·확장 프로그램 로컬 정보는 각 브라우저의 사이트 데이터 또는 확장 프로그램 데이터 삭제 기능으로 직접 제거할 수 있습니다.

12. 안전성 확보 조치

  • ZIP 분석 데이터와 서버 저장 신청 정보 분리
  • 자동화 목록의 확장 프로그램 로컬 저장과 최소 권한 사용
  • 공개 처리 함수의 허용 출처, 요청 크기, 형식과 필수 동의 검증
  • 관리자 API의 인증 토큰과 허용 이메일 이중 확인
  • 로그인·챌린지와 연속 DOM 오류 감지 시 자동화 중지
  • 관리자 화면의 검색 노출과 오프라인 캐시 차단
