문제 해결

분석이나 자동화가 되지 않을 때 확인하세요

ZIP 형식, 확장 프로그램 연결, Instagram 로그인 상태와 화면 언어를 순서대로 확인하면 대부분의 문제를 찾을 수 있습니다.

오류 메시지별 해결 방법

상황확인 방법
followers_*.json 또는 following.json을 찾지 못함Instagram 데이터 요청 시 파일 형식을 JSON으로 설정하고 ‘팔로워 및 팔로잉’을 포함해 다시 요청하세요.
ZIP 파일이 손상됨다운로드가 완전히 끝났는지 확인한 뒤 파일을 다시 내려받으세요.
확장 프로그램이 연결되지 않음Companion ZIP을 압축 해제해 Chrome 확장 프로그램으로 로드한 뒤 맞팔체커 페이지를 새로고침하세요.
팔로잉 버튼을 찾지 못함현재 페이지가 대상 프로필인지, Instagram 로그인이 유지되는지, 화면 언어가 지원 범위인지 확인하세요.
보안 확인·챌린지 감지작업은 자동 중지됩니다. Instagram 화면에서 확인 절차를 직접 완료한 뒤 충분히 쉬고 다시 시작하세요.
연속 오류로 중지됨Instagram UI가 변경됐을 수 있습니다. 실패 항목을 즉시 반복하지 말고 문의 페이지로 오류 문구를 보내 주세요.

어떤 목록을 자동화할 수 있나요?

현재 비맞팔과 A·B 시점 사이에 나를 언팔했지만 내가 현재도 팔로우 중인 계정을 만들 수 있습니다. 두 목록은 중복을 제거한 뒤 선택해서 전달합니다.

자동으로 바로 시작하나요?

아닙니다. 목록 전달만으로는 실행되지 않습니다. 사이드패널에서 최대 처리 수, 임의 대기, 휴식 시간을 확인하고 필수 확인란을 선택한 뒤 시작해야 합니다.

어떤 안전장치가 있나요?

1회 최대 30개, 최소 5초 이상 대기, 정기 휴식, 일시정지·중지, 로그인·챌린지 감지, 2회 연속 버튼 탐지 실패 시 자동 중지를 적용합니다.

작업 기록은 어디에 저장되나요?

웹 검토 상태와 자동화 큐·성공·실패 상태는 각각 현재 브라우저와 설치된 확장 프로그램의 로컬 저장소에 보관됩니다. 다른 기기로 자동 동기화되지 않습니다.

두 ZIP 비교는 어떻게 하나요?

분석 화면의 ‘관계 변화 비교’에서 이전 ZIP과 최신 ZIP을 선택하세요. 자동화 목록은 단순 팔로워 이탈 전체가 아니라, 이탈 계정 중 최신 시점에도 내가 팔로우 중인 계정만 추립니다.

자주 묻는 질문

ZIP 파일을 확장 프로그램이나 서버에 보내나요?

아니요. ZIP 분석은 웹 브라우저 안에서 수행됩니다. 확장 프로그램에는 선택한 Instagram 아이디 목록과 작업 상태만 전달됩니다.

비밀번호나 로그인 쿠키를 읽나요?

아니요. Companion은 Instagram에 이미 로그인된 탭에서 화면에 표시된 프로필 버튼을 찾으며 비밀번호, 인증 코드, 쿠키와 세션 토큰을 읽지 않습니다.

실패한 계정은 자동으로 계속 재시도하나요?

아니요. 실패로 남기고 연속 오류가 발생하면 중지합니다. 사용자가 원인을 확인한 뒤에만 ‘실패 다시 대기’를 눌러 재시도할 수 있습니다.

사이드패널을 닫으면 어떻게 되나요?

진행 중인 실행은 중지됩니다. 다음에 열면 처리 상태는 남아 있지만 자동으로 다시 시작하지 않습니다.

결과가 현재 Instagram과 다를 수 있나요?

ZIP은 생성 시점의 스냅샷입니다. Companion은 각 프로필에서 현재 팔로우 상태를 다시 확인하고 이미 팔로우하지 않는 계정은 건너뜁니다.

계속 해결되지 않는 경우

화면에 표시된 오류 코드와 사용한 Chrome·Instagram 화면 언어를 보내 주세요. ZIP 원본, 비밀번호, 인증 코드나 쿠키는 보내지 마세요.
