ZIP 분석과 선택 신청 정보는 서로 분리됩니다

ZIP은 웹 브라우저에서 읽고, 사용자가 선택한 자동화 아이디 목록은 설치한 Chrome Companion의 로컬 저장소로만 전달됩니다. 문의와 뉴스레터처럼 직접 제출한 정보만 별도의 서버 업무 시스템에 저장됩니다.

ZIP 원본 서버 전송 없음자동화 목록 확장 프로그램 로컬 저장비밀번호·쿠키 수집 없음

ZIP 분석 흐름

1사용자가 ZIP을 선택합니다Instagram에서 직접 내려받은 JSON ZIP을 현재 기기에서 선택합니다.
2브라우저가 관계를 계산합니다팔로워·팔로잉 JSON을 메모리에서 읽어 현재 비맞팔과 A·B 관계 변화를 계산합니다.
3결과를 화면에 표시합니다분석 결과는 맞팔체커 서버가 아니라 현재 브라우저 화면과 로컬 작업 기록에서 사용됩니다.

서버로 보내지 않는 정보

  • Instagram ZIP 원본과 JSON 원문
  • 분석된 사용자 아이디 전체 목록
  • 비맞팔·팔로워 이탈 결과 전체
  • 자동화 큐와 성공·실패 상태
  • Instagram 비밀번호·쿠키·인증 코드

웹 브라우저에 저장되는 정보

  • 작업공간 이름
  • 검토 완료 상태와 메모
  • 화면 설정과 마지막 사용 상태
  • 작업공간 연결용 비식별 해시 스케치

작업공간 관리나 브라우저 사이트 데이터 삭제 기능으로 제거할 수 있습니다.

Chrome Companion 자동화 흐름

1사용자가 대상 목록을 선택합니다현재 비맞팔 또는 팔로워 이탈 후에도 내가 팔로우 중인 계정 가운데 필요한 아이디만 체크합니다.
2아이디만 확장 프로그램에 전달합니다목록 이름, 출처, 아이디와 작업 상태를 Chrome 확장 프로그램 로컬 저장소에 저장합니다. ZIP 원본은 전달하지 않습니다.
3사용자가 사이드패널에서 실행합니다Instagram 프로필을 열고 화면의 팔로잉·팔로우 취소 버튼을 확인합니다. 로그인·챌린지·연속 오류가 감지되면 중지합니다.

Companion이 읽는 범위

  • 현재 Instagram 프로필 URL의 사용자 아이디
  • 화면에 표시된 버튼 텍스트와 대화상자
  • 확장 프로그램 로컬 큐와 실행 설정
  • 탭 로딩 상태

Companion이 읽지 않는 범위

  • Instagram 비밀번호와 인증 코드
  • 브라우저 쿠키와 세션 토큰
  • DM·게시물 내용·연락처
  • Instagram 비공개 API 응답
  • 선택하지 않은 분석 목록

문의 접수 정보

  • 이름과 답변 이메일
  • 문의 유형과 제목·내용
  • 개인정보 동의 기록
  • 접수·처리·이메일 알림 상태

뉴스레터·수요조사 정보

  • 신청 이메일과 동의 기록
  • 수신 해지 토큰과 발송 상태
  • 관심 기능과 희망 가격대
  • 추가 의견과 갱신 일시

서버에 저장되는 정보와 로컬 정보의 분리

문의, 뉴스레터, 수요조사와 캠페인 기록은 Supabase 기반 업무 데이터베이스에 분리해 저장합니다. ZIP 분석, 작업공간, 자동화 큐는 이 데이터베이스에 저장하지 않으며 서로 결합하지 않습니다.

분석과 Companion 자동화는 문의, 뉴스레터나 수요조사에 참여하지 않아도 사용할 수 있습니다.
