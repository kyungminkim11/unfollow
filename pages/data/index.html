ZIP과 팔로워·팔로잉 스캔 결과는 서버를 거치지 않습니다

공식 ZIP은 웹 브라우저에서 읽고, Instagram 웹 스캔 명단과 작업 상태는 Chrome Companion 안에서 처리합니다. 문의와 뉴스레터처럼 이용자가 직접 제출한 정보만 별도의 서버 업무 시스템에 저장됩니다.

ZIP 원본 서버 전송 없음관계 스캔 확장 프로그램 로컬 저장비밀번호·쿠키 수집 없음

공식 ZIP 분석 흐름

1사용자가 JSON ZIP을 선택합니다Instagram 계정 센터에서 직접 내려받은 파일을 현재 기기에서 선택합니다.
2웹 브라우저가 관계를 계산합니다팔로워·팔로잉 JSON을 메모리에서 읽어 현재 비맞팔과 A·B 관계 변화를 계산합니다.
3결과를 현재 브라우저에 표시합니다ZIP 원본과 관계 결과를 맞팔체커 서버로 업로드하지 않습니다.

Instagram 웹 스캔 흐름

1사용자가 Companion에서 스캔을 시작합니다로그인한 계정의 내 프로필인지 확인한 뒤 팔로워 목록 창을 엽니다.
2화면의 프로필 링크를 수집합니다팔로워와 팔로잉 목록을 차례로 스크롤하며 현재 화면에 로드되는 아이디를 읽습니다.
3맞팔 아닌 명단을 로컬에서 계산합니다팔로잉 명단에서 팔로워 명단을 빼고 결과를 확장 프로그램 저장소에 보관합니다.

Companion이 읽는 범위

  • 로그인한 계정의 내 프로필 경로
  • 팔로워·팔로잉 목록 창의 프로필 링크
  • 목록 스크롤과 로딩 상태
  • 팔로잉·팔로우 취소 버튼 텍스트
  • 확장 프로그램 로컬 설정과 작업 상태

Companion이 읽지 않는 범위

  • Instagram 비밀번호와 인증 코드
  • 브라우저 쿠키와 세션 토큰
  • DM·게시물 내용·연락처
  • Instagram 비공개 API 응답
  • 다른 사람 프로필의 전체 관계 명단

확장 프로그램에 저장되는 정보

  • 내 Instagram 아이디
  • 팔로워·팔로잉·맞팔 아닌 아이디
  • 스캔 완료 여부, 시각과 경고
  • 선택한 팔로우 취소 작업 목록
  • 성공·실패·건너뜀 상태와 실행 설정

웹 브라우저에 저장되는 정보

  • ZIP 작업공간 이름
  • 검토 완료 상태와 메모
  • 화면 설정과 마지막 사용 상태
  • 작업공간 연결용 비식별 해시 스케치

맞팔 아닌 명단과 작업 목록

1명단을 먼저 확인합니다웹 또는 Companion에서 팔로워, 팔로잉과 맞팔 아닌 명단을 확인합니다.
2사용자가 필요한 아이디를 선택합니다명단 생성만으로는 팔로우 상태가 바뀌지 않습니다.
3별도 확인 후 작업을 시작합니다Companion 사이드패널에서 처리 수와 대기 시간을 정하고 시작합니다.

서버로 보내지 않는 정보

  • Instagram ZIP 원본과 JSON 원문
  • 팔로워·팔로잉·맞팔 아닌 전체 명단
  • 자동화 큐와 성공·실패 상태
  • 작업공간 메모와 분석 결과
  • Instagram 로그인 정보

서버에 저장되는 업무 정보

  • 문의자가 제출한 이름·이메일·내용
  • 뉴스레터 이메일과 동의 기록
  • 프리미엄 수요조사 답변
  • 뉴스레터 캠페인과 발송 상태
  • 관리자 감사 기록

로컬 정보와 서버 정보의 분리

문의, 뉴스레터, 수요조사와 캠페인 기록은 Supabase 기반 업무 데이터베이스에 분리해 저장합니다. ZIP 분석, 웹 작업공간, 관계 스캔 명단과 팔로우 취소 작업 목록은 이 데이터베이스에 저장하지 않고 서로 결합하지 않습니다.

분석과 관계 스캔은 문의, 뉴스레터 또는 수요조사에 참여하지 않아도 사용할 수 있습니다.
