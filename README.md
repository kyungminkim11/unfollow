# 맞팔체커

Instagram에서 내려받은 관계 데이터 ZIP을 브라우저에서 직접 분석하는 로컬 웹 서비스입니다.
로그인 정보나 비밀번호를 요구하지 않으며, 선택한 ZIP과 분석 결과를 서버로 전송하지 않습니다.

- 서비스: https://unfollow.lavalabs.co.kr
- 운영: Lava Labs
- 지원 데이터: Instagram JSON 형식의 팔로워 및 팔로잉 ZIP
- 자동 언팔: 제공하지 않음
- 현재 운영: 무료 베타

## v21 서비스 구성

- **분석**: 맞팔, 나만 팔로우 중, 나를 팔로우 중인 계정 분류
- **관계 변화 비교**: 이전 ZIP과 최신 ZIP의 팔로워·팔로잉·맞팔 변화 비교
- **작업공간**: 브라우저 로컬 저장, 이름 변경, 개별·전체 백업 및 삭제
- **결과 활용**: 검색, 상태 기록, 아이디 복사, CSV 저장
- **모바일 앱 UI**: 네이티브형 앱바, 작업 중심 첫 화면, 안전 영역을 반영한 하단 탭바와 터치 인터랙션
- **연결 상태 안내**: 실제 네트워크 확인이 연속으로 실패한 경우에만 작고 닫을 수 있는 오프라인 모드 안내 표시
- **문의 센터**: 이름, 이메일, 문의 유형과 내용을 서버에서 검증해 저장하고 운영자에게 즉시 이메일 알림
- **문의 관리**: 관리자 화면에서 문의 검색, 상태 변경, 이메일 답장과 삭제
- **희망자 뉴스레터**: 회원가입 없이 프리미엄 출시와 중요 업데이트 이메일 수신 신청
- **신청 알림**: 신규 문의와 뉴스레터 신청을 운영자 이메일로 즉시 알림
- **수신 해지**: 이메일 입력 또는 메일별 임의 토큰 링크로 구독 정보와 동의 기록 삭제
- **프리미엄 수요조사**: 희망 기능, 가격대, 운영 계정 수와 추가 의견 수집·갱신·삭제
- **관리자 대시보드**: 이메일 매직링크 인증, 구독자·문의·수요 통계, 검색, CSV, 상태 관리와 삭제
- **캠페인 운영**: 뉴스레터 초안, 실시간 미리보기, 대상 세그먼트, 예약·즉시 대기열, 발송 진행률, 재시도와 취소
- **정책 페이지**: 개인정보 처리방침, 이용약관, 데이터 처리 안내, 프리미엄 예정 기능
- **접근성·반응형**: 데스크톱·태블릿·모바일, 다크모드, 키보드 탐색 지원
- **오프라인 사용**: 설치형 웹앱(PWA)과 일반 페이지별 오프라인 캐시 지원. 관리자 화면은 네트워크 전용

## 개인정보와 안전

ZIP 파일은 사용자의 브라우저 메모리에서만 읽습니다. 작업 기록과 설정은 현재 브라우저의 로컬 저장소에 보관되며 사용자가 삭제하거나 백업할 수 있습니다. 작업공간 연결에는 사용자 아이디 원문 대신 브라우저에서 계산한 비식별 해시 스케치만 저장됩니다.

문의 폼을 제출하면 이름, 답변 이메일, 문의 유형, 제목, 문의 내용과 동의 기록을 Supabase 서울 리전 데이터베이스에 저장합니다. 신규 문의는 운영자에게 이메일로 알리고 관리자 화면에서 처리 상태를 관리합니다. 뉴스레터를 신청한 경우에는 이메일과 동의 기록만 별도 저장합니다. 프리미엄 수요조사에 참여한 경우 이메일, 선택 기능, 희망 가격, 계정 수 범위와 추가 의견을 별도 테이블에 저장합니다. 이 정보들은 ZIP 원본·분석 결과와 결합하지 않습니다.

캠페인을 대기열에 넣을 때에는 발송 시점의 활성 구독자 이메일과 수신 해지 토큰을 캠페인별 전달 목록에 스냅샷으로 저장합니다. 캠페인을 삭제하면 전달 목록도 함께 삭제됩니다. 실제 메일 발송이 활성화되면 수신 이메일, 제목과 본문이 발송 제공자인 Resend로 전달됩니다.

공개 브라우저에서는 데이터베이스를 직접 읽거나 쓸 수 없습니다. 공개 신청 함수는 허용 Origin, 요청 크기, 이메일 형식, 필수 동의, 제출 시간과 허니팟을 검증합니다. 관리자 API는 유효한 Supabase 사용자 JWT와 관리자 허용 이메일을 모두 확인합니다.

맞팔체커는 Instagram 또는 Meta와 제휴하거나 공식적으로 운영되는 서비스가 아닙니다. Instagram 화면에서 직접 팔로우를 변경하기 전에는 계정 상태와 변경 대상을 다시 확인해야 합니다.

## 무료 베타와 유료화

현재 결제되는 유료 상품과 자동 갱신 구독은 없습니다. 향후 프리미엄은 분석 결과를 가리는 방식보다 분석 이력, 여러 계정, 고급 비교, 리포트, 기기 동기화 같은 관리 기능을 중심으로 검토합니다. 수요조사 결과는 개발 우선순위와 가격 검토 자료이며 확정된 계약 조건이 아닙니다.

실제 판매 전에는 가격, 이용 기간, 환불, 자동 갱신 여부, 회원가입과 클라우드 저장 항목을 별도로 고지합니다.

## 이메일 알림과 뉴스레터 발송 설정

운영자 알림은 Formspree를 기본 폴백으로 사용하므로 Resend가 연결되지 않은 상태에서도 신규 문의와 뉴스레터 신청 알림을 받을 수 있습니다.

문의자 접수 확인 메일, 뉴스레터 신청 확인 메일과 실제 캠페인 발송에는 Supabase Edge Function 환경변수가 필요합니다.

```text
RESEND_API_KEY
RESEND_FROM_EMAIL
RESEND_REPLY_TO       # 선택, 기본 info@lavalabs.co.kr
CONTACT_NOTIFY_EMAIL  # 선택, 기본 lavalabs.ceo@gmail.com
FORMSPREE_ENDPOINT    # 선택, 운영자 알림 폴백
```

Resend 연결이 없으면 관리자 화면에 `연결 필요`로 표시되며 실제 뉴스레터 캠페인은 전송되지 않습니다. 캠페인 발송은 50건 단위 배치로 처리하고, 캠페인별 성공·실패·남은 건수를 저장합니다. 문의와 구독 신청 자체는 정상 저장되며 운영자 알림은 Formspree 폴백으로 전달됩니다.

## 빌드와 검증

`main` 브랜치에 반영되면 GitHub Actions가 정적 사이트를 생성하고 다음 항목을 검사합니다.

- ZIP 용량·파일 수·압축률·JSON 크기 안전 제한
- 기존 분석, 작업 기록과 두 시점 비교 기능
- 1536·1280·900·390·320px 레이아웃
- 모바일 앱바, ZIP 선택 위치, 하단 탭바와 터치 영역
- 정상 연결 상태에서 오프라인 안내가 표시되지 않는지 확인
- 연결 실패 시 안내가 콘텐츠와 하단 탭바를 가리지 않고 닫히는지 확인
- Axe 기반 critical/serious 접근성 오류
- 문의 입력 검증, 접수번호와 운영자 알림
- 뉴스레터 신청·이메일 해지와 토큰 해지 입력 검증
- 프리미엄 수요조사 저장·갱신·삭제
- 관리자 API 비로그인 차단
- 문의 목록·상태 관리와 캠페인 초안·미리보기·대상 계산
- 관리자 페이지 검색 차단과 오프라인 캐시 제외

## Supabase 백엔드

### 공개 문의

- Edge Function: `unfollow-contact`
- Table: `unfollow_inquiries`
- 신규 문의 저장, 접수번호 생성, 운영자 알림, 선택적 문의자 확인 메일

### 공개 뉴스레터

- Edge Function: `unfollow-newsletter`
- Tables: `unfollow_newsletter_subscribers`, `unfollow_newsletter_consent_events`
- 구독자별 `unsubscribe_token`으로 이메일 주소를 URL에 포함하지 않는 해지 지원
- 신청자 확인 메일과 운영자 신규 신청 알림 상태 기록

### 프리미엄 수요조사

- Edge Function: `unfollow-premium-interest`
- Table: `unfollow_premium_interest`

### 관리자와 캠페인

- Edge Functions: `unfollow-newsletter-admin`, `unfollow-newsletter-campaigns` (`verify_jwt=true`)
- Tables: `unfollow_admin_allowlist`, `unfollow_admin_audit_log`, `unfollow_newsletter_campaigns`, `unfollow_newsletter_campaign_deliveries`
- 로그인: Supabase Auth 이메일 매직링크
- 현재 허용 관리자: `lavalabs.ceo@gmail.com`

모든 업무 테이블은 RLS를 사용하고 공개 직접 접근 정책을 두지 않습니다. 공개 신청은 검증된 Edge Function으로만 처리하며 관리자 조회·삭제·상태 변경·발송은 유효한 JWT와 허용 이메일 확인을 모두 통과해야 합니다.
